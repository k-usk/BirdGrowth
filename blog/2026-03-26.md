# WidgetKit再構築：AppGroupによる確実な同期とプレミアムなデザインの追求

## 作業内容
本日はウィジェット機能のゼロベースでの再構築を実施した。主な目的は、小・大（Small & Large）サイズのフルサポート、HealthKitから取得した実歩数のリアルタイムなデータ反映、およびApp Groupによるデータ同期の安定化である。

実装においては、`HealthKitManager` で取得した最新の歩数データを `WidgetDataManager` を介して `UserDefaults(suiteName:)` に保存し、ウィジェット側から即座に参照できる基盤を確立した。

## 苦労した点
最も困難だったのは、OS側の挙動（Redacted表示）とApp Group間のデータ同期タイミングの不一致である。当初、ウィジェット側でデータが取得できずプレースホルダが表示され続ける問題が発生した。

これに対し、まずダミーデータを用いた視覚的デバッグを行い、次に `WidgetCenter.shared.reloadAllTimelines()` の呼び出し箇所（ViewModelの保存処理直後）を修正することで、意図したタイミングでの更新を可能にした。また、イラスト表示においてスプライトシートから適切な領域を正確に切り出し（`.clipped()`）、それを角丸のカード（`RoundedRectangle`）内に収めるためのレイアウト調整にも細心の注意を払った。特に、サイズの大きなイラストがレイアウトの垂直マージンを押し潰してしまう問題は、イラストサイズを `240pt` から `210pt` 付近へ最適化し、パディングを再計算することで解決した。

## コードの要所抜粋とその解説

### 1. App Groupによるデータ共有の抽象化
データ共有のロジックを `WidgetDataManager` に集約し、UserDefaultsへの保存とWidgetCenterへの更新通知を一連のフローとしてカプセル化した。これにより、ViewModel側は共有の詳細を意識することなくデータを永続化できる。

```swift
struct WidgetDataManager {
    static let shared = WidgetDataManager()
    private let suiteName = "group.com.cluppou.BirdGrowth"
    
    func save(steps: Int, birdName: String, spriteFileName: String, stageIndex: Int, statusMessage: String) {
        guard let defaults = UserDefaults(suiteName: suiteName) else { return }
        defaults.set(steps, forKey: "steps")
        // ... 他のプロパティの保存
        WidgetCenter.shared.reloadAllTimelines()
    }
}
```

### 2. 成長段階に連動したセグメントカラー
本体アプリの `GrowthTrailView` の仕様に基づき、ウィジェット側でも成長段階（卵・ひな・成鳥）に応じて進捗バーの色を動的に変更するロジックを組み込んだ。

```swift
let activeColor: Color = {
    if entry.stageIndex == 0 { return .white } // 卵
    if entry.stageIndex == 1 { return .orange.opacity(0.6) } // ひな
    return Color(red: 1.0, green: 0.7, blue: 0.7) // 成鳥（ピンク）
}()

Capsule()
    .fill(reached ? activeColor : Color.brown.opacity(0.1))
    .frame(height: 3)
```

この実装により、ユーザーはウィジェットを一瞥するだけで現在の鳥の状態と歩数目標への進捗を直感的に把握することが可能となった。MVVMパターンを遵守し、ロジックを適切に分散させたことで、保守性の高いウィジェット実装を達成した。

## 技術的な備忘録：Xcodeにおける環境構築手順

ウィジェットおよびHealthKitを正常に動作させるためには、コードの実装だけでなくXcode上のプロジェクト設定が不可欠である。以下にその手順をまとめる。

### 1. Signing & Capabilities の設定
メインアプリターゲットおよびウィジェット拡張ターゲットの両方で、以下の能力（Capabilities）を有効にする必要がある。

- **App Groups**: 
  - メインアプリとウィジェットの両方で有効化する。
  - 同じグループID（例: `group.com.cluppou.BirdGrowth`）を指定する。これにより、異なるプロセス間でのUserDefaults共有が可能になる。
- **HealthKit** (メインアプリのみ):
  - 歩数データへのアクセスを許可するために有効化する。

### 2. Info.plist の更新
ユーザーに対してプライバシーへの同意を求めるため、以下のキーを Info.plist に追加する必要がある。

- **NSHealthShareUsageDescription**:
  - 理由を説明する文字列（例：「歩数を取得して、鳥の成長に反映させるために使用します。」）を記述する。

### 3. Widget Extension の追加
Xcodeの「File > New > Target...」から「Widget Extension」を追加する。この際、「Include Configuration Intent」のチェックは今回のようなシンプルな静的ウィジェットの場合は不要である。

これらの設定が一つでも欠けると、シミュレータでは動作しても実機でデータが表示されない等のトラブルに直結するため、開発環境の初期構築時には必ず確認すべき項目である。
