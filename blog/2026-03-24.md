# MVVM移行と220通りの動的メッセージ群による育成体験の深化

本日は、BirdGrowthのコードベースをMVVMアーキテクチャへ刷新し、さらに1,000歩単位のきめ細やかな進捗表現と、220パターンの動的メッセージシステムを実装した。これにより、保守性の向上と「鳥の命」を感じさせるナラティブな演出を両立させた。

## 作業内容

1.  **MVVMアーキテクチャへの完全移行**
    - `ContentView.swift` に集中していたビジネスロジック、状態管理、サブViewを `Models`, `ViewModels`, `Components` フォルダへ分離。
    - iOS 17以降推奨の `@Observable` マクロを採用し、Viewとロジックを疎結合に再構築。
2.  **10段階オーガニック・セグメント・ゲージの実装**
    - SF Symbols (`feather.fill`) の互換性問題を解消するため、SwiftUIの `Capsule` 図形を用いたカスタムゲージを開発。
    - 1,000歩単位で10段階の進捗を表示し、成長段階（卵・ひな・成鳥）に合わせてゲージの色が白からオレンジ、そして優しいピンクへと変化する動的な色彩設計を導入。
3.  **220パターンの「鳥のさえずり」バンク構築**
    - 0歩から10,000歩超まで、1,000歩ごとの各セグメントに20種類ずつのメッセージ（計220案）を用意。
    - 成長に合わせたナラティブなセリフに加え、ユーモアや達成後の特別メッセージを抽選するロジックをViewModelに実装。

## 苦労した点

単なる「歩数計アプリ」から「育成アプリ」への昇華を目指し、200通り以上のメッセージにナラティブな一貫性を持たせることに注力した。18文字以内という厳しい制約の中で、卵の時の微かな胎動から、成鳥になった時の深い信頼関係、時折混ざるユーモアを表現するのはクリエイティブな挑戦であった。

また、SF Symbolsのバージョン依存による表示エラー（feather.fill非対応）を機に、外部アセットに頼らずSwiftUIの標準図形のみで洗練されたゲージを構築するアプローチへ舵を切った。結果として、描画負荷が低く、かつ全てのiOSバージョンで安定して動作する堅牢なUIを実現できた。

## コードの要所抜粋とその解説

### 1. @Observable によるクリーンな状態管理
従来の `@State` による複雑な計算プロパティを ViewModel へ集約した。

```swift
@Observable
class BirdViewModel {
    var steps: Int = 0 {
        didSet { updateMessageIfNeeded() }
    }
    
    // 1,000歩の境界を跨いだ時のみ新しいメッセージを抽選
    private func updateMessageIfNeeded() {
        let segmentIndex = min(max(steps / 1000, 0), 10)
        if segmentIndex != lastSegmentIndex {
            currentMessage = BirdMessageBank.messages[segmentIndex].randomElement() ?? "……"
            lastSegmentIndex = segmentIndex
        }
    }
}
```

**解説**: 
`steps / 1000` を用いてセグメントインデックスを算出。歩数が更新されるたびに再抽選するのではなく、境界線を越えたタイミングでのみ抽選を行うことで、同じセグメント内ではメッセージが固定され（しかし次回はランダム）、歩行への報酬としての「新しい一言」を感じられる設計にしている。

### 2. ステージ連動型オーガニック・ゲージ
Viewは受け取った `stage` 情報から、自身の色を決定する。

```swift
private func segmentView(for index: Int) -> some View {
    let activeColor: Color = {
        switch stage {
        case .egg: return .white
        case .chick: return .orange.opacity(0.6)
        case .adult: return Color(red: 1.0, green: 0.7, blue: 0.7) // 優しいピンク
        }
    }()
    
    return Capsule()
        .fill(reached ? activeColor : Color.brown.opacity(0.1))
        .frame(height: 4)
}
```

**解説**: 
単一のProgressViewを使用せず、10個の `Capsule` を並べてセグメント化することで、「一歩ずつ、確実に目標へ近づいている」という手触り感を与えている。色は背景のパステルグラデーションに合わせて調整し、成鳥時は特に「温かみのあるピンク」にすることで、ゴール時の達成感をトーンダウンさせず、かつ目に優しいハーモニーを実現した。

## まとめ

今回のリファクタリングにより、データの流れ（ViewModel）と表示（View）が明確に分離され、今後の「新しいスプライトの追加」や「育成ロジックの複雑化」にも耐えうる土台が整った。220通りのメッセージは、ユーザーと鳥との間の「共通の記憶」となり、歩行という日常の動作を愛おしい体験へと変える大きな一歩となった。
