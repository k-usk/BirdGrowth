# Xcodeプロジェクトの初期構築とGit管理基盤のセットアップ

本日は、のちにオキナインコ育成アプリとなる「BirdGrowth」プロジェクトの立ち上げ日である。iOSアプリ開発のベースラインとなる環境構築、およびソースコード管理のための初期コミットを行った。

## 本日の作業内容

1. **Xcodeプロジェクトの新規作成**
2. **Gitリポジトリの初期化とファイル除外設定（.gitignore）**
3. **初期アセット・構成ファイルの登録**

## 1. プロジェクトの初期化
Xcodeを用いて「BirdGrowth」という新規プロジェクトを生成した。インターフェースとして `SwiftUI` を選択し、モダンな宣言的UIを用いた構成を基本としている。

## 2. バージョン管理のセットアップ

Gitを用いたソースコード管理を開始するにあたり、不要なファイルがリポジトリに混入するのを防ぐため `.gitignore` を設定した。

```gitignore
# .gitignore (主要部分の抜粋)
## User settings
xcuserdata/

## App packaging
*.ipa
*.dSYM.zip

## Build and dependencies
.build/
Carthage/Build/
Pods/
```

**解説:**
iOS開発において、ビルドの中間ファイル（`.build/`）増や開発者固有のウィンドウ設定（`xcuserdata/`）などをそのままGitに含めると、コンフリクト（競合）の原因となりリポジトリの容量も無駄に消費する。そのため、プロジェクトの初期段階で必ず `.gitignore` を配置し、純粋なソースコードのみを管理する体制を整える必要がある。今回はSwift/Objective-Cの標準的なignoreテンプレートを活用している。

本日の作業により、何があっても元に戻せる「セーブポイント」の基盤が用意された。明日からはこの土台の上で、本格的なUIの実装に着手していく。
