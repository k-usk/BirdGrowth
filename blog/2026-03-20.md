# SwiftUIによる歩数連動型育成UIの基礎実装とAI開発指針の定義

本日は、オキナインコを育てるiOSアプリ「BirdGrowth」の実質的な開発初日として、SwiftUIを用いたメイン画面の基礎UI構築と、歩数（Steps）に応じた成長ロジックの実装を行った。また、今後のAIアシスタント（Cursor等）を用いた開発効率化を見据え、プロジェクト固有の開発ガイドラインを定義した。

## 本日の作業内容

1. **AI開発アシスタント用ルールファイル（.cursorrules）の作成**
2. **ContentViewにおける歩数連動の成長ロジック実装**
3. **SF Symbolsを活用したプロトタイプデザインの構築**

## 1. 開発指針（.cursorrules）の策定

持続可能かつ一貫性のあるAIとの協業体制を築くため、プロジェクトルートに `.cursorrules` を配置した。

```text
# .cursorrules (抜粋)
- 回答や説明はすべて「日本語」で行ってください。
- 複雑なライブラリは避け、SwiftUIの標準機能を優先して使用してください。
- コードを全書き換えするのではなく、必要な箇所（差分）だけを提示するようにしてください。
```

**解説:**
個人開発においてはAIのコンテキスト（推論の前提）をいかにブレさせないかが重要である。「日本語での回答」「差分のみの提示（クレジット節約）」「SwiftUI標準機能の優先」を明文化することで、AIが無駄に複雑な外部ライブラリを導入したり、ファイル全体を不要に書き換えたりする事故を未然に防ぐことができる。

## 2. 歩数連動型ステージロジックの実装

歩数（`steps`）に応じて「卵 → ヒナ → 成鳥」へと状態が変化するコアロジックを実装した。状態管理には `@State` を用い、列挙型（`enum`）で成長段階を定義している。

```swift
// ContentView.swift (抜粋)
@State private var steps = 0

private enum GrowthStage {
    case egg
    case chick
    case adult
}

private var stage: GrowthStage {
    switch steps {
    case 0 ... 999:
        return .egg
    case 1000 ... 4999:
        return .chick
    default:
        return .adult
    }
}
```

**解説:**
View内で計算プロパティ（Computed Property）として `stage` を定義することで、`steps` の値が変化するたびに自動的に現状のステージが再評価され、UIに即座に反映される。この宣言的な画面更新こそがSwiftUIの最大の強みである。

## 3. UIの構築とデザイン調整

アプリのコンセプトである「優しく清潔感のあるブルー」を基調に、画面全体のレイアウトとデバッグ用の操作ボタンを配置した。

```swift
// 背景グラデーションの設定
LinearGradient(
    colors: [
        Color.blue.opacity(0.12),
        Color.cyan.opacity(0.16),
        Color.blue.opacity(0.10),
    ],
    startPoint: .topLeading,
    endPoint: .bottomTrailing
)
.ignoresSafeArea()
```

インコの成長段階を示すアイコンには、標準で用意されているSF Symbols（`egg`, `bird`, `bird.fill`）を採用し、ステージごとに色（cyan, blue, indigo）を切り替える実装を行った。これにより、外部画像アセットを用意することなく、短時間で質の高いプロトタイプを構築することができた。

開発初日としては、状態管理とUIのバインディングというSwiftUIの基礎設計が綺麗に定まったため、今後はこの基盤を拡張していく形となる。
